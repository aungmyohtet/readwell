# ReadWell V2 Launch PRD

## Product Direction

ReadWell v2 should make the current learning loop more decisive, not more complicated.
The product already has the right base: graded stories, grammar in context, practice, quizzes, and adaptive review.
V2 should improve three moments:

1. starting in the right place
2. knowing what to do after each chapter
3. finding the right next story or review target

## Product Goals

1. Help learners choose an appropriate level and first story.
2. Shift progress language from simple completion to mastery and review readiness.
3. Reduce wasted effort by allowing focused review and retry.
4. Make the library browsable by learning need, not only by story title.
5. Prepare the app for launch decisions with basic product analytics.

## Target Users

1. Independent learners aged 10-18
2. Parents supporting regular English study at home
3. Teachers who want structured grammar-reading practice

## Core V2 Feature Set

### Must Have

1. Placement check and starting recommendation
2. Mastery-based chapter states
3. Focused retry mode for missed items and weak grammar
4. Better end-of-chapter next-step guidance
5. Browse filters by grammar, topic, and review need
6. Basic analytics for launch decisions

### Should Have

1. Grammar unit bundles
2. Quiet weekly goals and streaks
3. Parent or teacher progress summaries
4. Reader bookmarks and saved examples
5. Stronger onboarding around the chapter study loop

### Later

1. Assignable reading paths
2. Cohort-level teacher dashboards
3. Audio support
4. Internal content QA dashboard
5. Broader recommendation logic across mastery patterns

## Success Metrics

1. Higher first-session chapter completion rate
2. High placement completion rate for new users
3. Increased review-queue engagement
4. Increased retry usage after low quiz performance
5. Better 7-day return rate
6. Clearer movement from completed chapters to stable/mastered chapters

## Scope Boundaries

### In Scope

1. Placement recommendation
2. Mastery and review states
3. Focused retry flow
4. Smarter completion screen
5. Browse by learning need
6. Analytics instrumentation

### Out Of Scope

1. Runtime AI features
2. Open-ended writing evaluation
3. Social or multiplayer features
4. Heavy gamification
5. Full teacher classroom management tools

## Key User Stories

1. As a new learner, I want a short placement check so I know where to begin.
2. As a learner, I want to see whether a chapter is stable, weak, or mastered.
3. As a learner, I want to retry only what I missed.
4. As a learner, I want a clear next action after every chapter.
5. As a learner, I want to browse by grammar target or topic.
6. As a parent or teacher, I want a concise summary of progress and weak areas.

## Recommended Launch Cut

If scope must stay tight, v2 launch should include:

1. Placement check
2. Mastery-based progression
3. Focused retry mode
4. Improved completion and next-step guidance
5. Browse filters by grammar, topic, and review status
6. Basic analytics
